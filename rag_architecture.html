0
环境准备（一次性）
运行 download_model.py，通过 HuggingFace Hub 下载所有预训练模型到本地 pre_train_model/ 目录：
m3e-large（向量检索）、bge-m3（向量检索）、bce-reranker-base_v1（重排）、text2vec-base-chinese（评分）
注意：三个大语言模型（Qwen2/Baichuan2/ChatGLM3）需单独获取，放置于 models/ 目录
download_model.py huggingface_hub.snapshot_download
1
知识库构建（首次自动触发）
当向量库不存在时，系统自动从原始 PDF 构建知识库：
• ParseBlock(1024/512)：pdfplumber 识别字体大小变化切块，保证小标题+内容完整性
• ParseAllPage(256/512)：PyPDF2 全文提取后按句号分割，滑动窗口保证跨页语义连续
• ParseOnePageWithRule(256/512)：PyPDF2 逐页提取，超长段落按句号均匀切分
→ 6次解析结果去重合并 → 向量化 → 分别建立 FAISS 索引（M3E/BGE）和内存索引（BM25/TF-IDF）
pdf_parse.DataProcess FAISS.from_documents BM25Retriever / TFIDFRetriever
2
Query 增强（可选，prompt_enhance=True）
先用大模型对原始问题进行简短回答（限16字），将回答拼接到原始问题中，形成增强检索词。
原理：LLM先验知识帮助扩充语义，提升向量召回的相关性，尤其改善简短或模糊问题的检索效果
retriever_query = query + LLM.infer("请简洁地回答...{query}")
ChatLLM.infer / ChatGPTProxy.infer Prompt Engineering
3
四路并行召回（各取 Top-15）
使用增强后的 retriever_query 同时触发四路检索，覆盖不同类型的相关性：
• M3E 语义召回：m3e-large 嵌入 + FAISS L2 距离，擅长语义相似度匹配
• BGE 语义召回：bge-m3 嵌入 + FAISS L2归一化，擅长跨语言和专业领域
• BM25 关键词召回：jieba 分词 + BM25 词频统计，擅长精确关键词命中
• TF-IDF 关键词召回：jieba 分词 + TF-IDF 权重，擅长重要词汇加权匹配
四路合计最多 60 条候选文档
M3eRetriever.GetTopK(15) BgeRetriever.GetTopK(15) Bm25Retriever.GetBM25TopK(15) TfidfRetriever.GetBM25TopK(15)
4
Cross-Encoder 精排（多路融合 → Top-6）
将60条候选文档合并，通过 reRankLLM 进行精排：
• 构建 (query, doc) 文本对，批量输入交叉编码器
• 模型输出每对的相关性分数（logits），按分数降序排列
• 取前 Top-K（默认6）条最相关文档
优势：Cross-Encoder 比 Bi-Encoder 精度更高，因为它同时看到 query 和 doc，能捕获更细粒度的语义交互
reRankLLM.predict(query, all_docs) BCE / BGE Cross-Encoder
5
Prompt 构建（5路并行）
同一问题构建5种不同的 Prompt，用于对比评测：
• answer_1：多路召回 + Cross-Encoder 重排后 Top-6 → mutil_rerank_inputs
• answer_2：仅 M3E 单路 Top-6 → m3e_inputs（含距离分数，用于兜底过滤）
• answer_3：仅 BGE 单路 Top-6 → bge_inputs（含距离分数）
• answer_4：仅 BM25 单路 Top-6 → bm25_inputs
• answer_5：仅 TF-IDF 单路 Top-6 → tfidf_inputs
Prompt 模板：基于已知信息回答，无答案时说"无答案"，使用中文
get_prompt_template(docs, query) get_emb_docs / get_distribute_docs / get_emb_distribute_rerank
6
大模型 Batch 推理
将5条 Prompt 组成 batch，一次性调用 LLM 推理：
• 本地推理（ChatLLM）：逐条生成，max_new_tokens=2000，greedy decoding（temperature=0）
• API 推理（ChatGPTProxy）：ThreadPoolExecutor 并行调用 HuggingFace Router API
根据 model_name 是否含 "Qwen" 字符串自动切换推理路径
ChatLLM.infer(batch_input) ChatGPTProxy.infer(batch_input)
7
兜底过滤 + 保存 + 评测
• 兜底过滤：若 M3E 或 BGE 最小向量距离 > 500，说明知识库中无相关内容，强制输出"无答案"
• 保存结果：每条问题保存 5-7 个答案字段到 result.json
• 评分评测：使用 text2vec-base-chinese 计算预测与标准答案的语义相似度，输出 metrics.json
设计价值：5路对比答案让研究者可以直观比较不同召回策略的效果差异
距离阈值过滤（score > 500 → 无答案） test_score.test_metrics result.json / metrics.json

run.py
主程序入口 / CLI 控制器
  • 使用 argparse 定义所有可配置参数
  • 支持 LLM 选型（qwen2/baichuan2/chatglm3/Qwen API）
  • 支持 Reranker 选型（bce/bge）
  • 串联推理（question_test）和评测（test_metrics）
依赖： config.py generate_answer test_score
⚙️
generate_answer.py
RAG 核心流程调度器
  • question_test：批量 RAG 推理主函数
  • get_prompt_template：构建中文 RAG Prompt
  • get_emb_docs：向量召回 Prompt 构建 + 距离过滤
  • get_distribute_docs：稀疏召回 Prompt 构建
  • get_emb_distribute_rerank：多路合并重排 Prompt 构建
依赖： hf_model huggingface_proxy rerank_model 4x retriever
📄
pdf_parse.py
多策略 PDF 解析器
  • ParseBlock：分块法（pdfplumber），保证标题+内容完整
  • ParseAllPage：滑窗法（PyPDF2），保证跨页语义连续
  • ParseOnePageWithRule：规则法（PyPDF2），均匀切分
  • SlidingWindow：滑动窗口核心算法
  • Datafilter：数据清洗（多级分割去噪）
依赖： pdfplumber PyPDF2
🔷
retriever/m3e_retriever.py
M3E 稠密语义向量召回
  • 使用 moka-ai/m3e-large 模型进行文本向量化
  • FAISS 向量数据库存储（batch_size=64）
  • GetTopK：返回 (Document, L2_score) 列表
  • 首次运行自动构建并保存索引
依赖： langchain FAISS pdf_parse
🔶
retriever/bge_retriever.py
BGE 稠密语义向量召回
  • 使用 BAAI/bge-m3 模型（L2归一化）
  • HuggingFaceBgeEmbeddings + FAISS 索引
  • normalize_embeddings=True，提升相似度准确性
  • 与 M3E 互补，覆盖更多语义场景
依赖： langchain FAISS pdf_parse
📝
retriever/bm25_retriever.py
BM25 关键词稀疏召回
  • jieba 中文分词，构建 BM25 索引
  • 适合精确关键词匹配（人名、地名、术语）
  • ⚠️ 当前 GetBM25TopK 有 break，实际只返回1条
  • 内存存储，无持久化
依赖： jieba langchain BM25Retriever pdf_parse
📊
retriever/tfidf_retriever.py
TF-IDF 关键词稀疏召回
  • jieba 中文分词，TF-IDF 权重加权
  • 与 BM25 互补，对重要词汇有更强区分度
  • GetBM25TopK 无 break，返回全部 top_k 条
  • 内存存储，无持久化
依赖： jieba langchain TFIDFRetriever pdf_parse
📐
rerank_model.py
Cross-Encoder 重排序模型
  • 支持 BCE 和 BGE 两种 Reranker
  • predict：构建 (query, doc) 文本对批量打分
  • 模型转 fp16 节省显存
  • 推理后自动释放 GPU 显存（torch_gc）
依赖： transformers config.py
💾
hf_model.py
本地 HuggingFace LLM 推理
  • 支持 Qwen2 / Baichuan2 / ChatGLM3 三种模型
  • float16 加载，device_map="auto" 自动分配 GPU
  • infer：批量推理，自动拼接对应 Chat Template
  • max_new_tokens=2000, greedy decoding
依赖： transformers config.py
☁️
huggingface_proxy.py
HuggingFace API 代理
  • 通过 HuggingFace Inference Router API 调用云端 LLM
  • OpenAI 兼容客户端，无缝对接
  • ThreadPoolExecutor 并行推理，高效处理 batch
  • 从 .env 读取 HF_TOKEN
依赖： openai python-dotenv
📈
test_score.py
RAG 评测评分
  • 加载 text2vec-base-chinese 语义相似度模型
  • 对比预测答案与标准答案，计算语义相似度
  • 对5种召回策略分别评分，便于横向对比
  • 结果写入 metrics.json
依赖： text2vec config.py
🌐
benchmark/model_serve.py
FastAPI 模型服务
  • FastAPI + uvicorn 提供 REST API 服务
  • POST /qwen 接口，接收 message 列表
  • bfloat16 加载 Qwen2-7B-Instruct
  • 启动：uvicorn model_serve:app --port 8000
依赖： fastapi uvicorn transformers
